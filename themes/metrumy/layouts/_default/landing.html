Hi, this is how I work. It's about coding to sell.

There are many factors that make a product page design sell better. But there's only a few that repetitively give big impact.

Smart image loading

On the beginning only the most important pictures are being downloaded. That way they will show up faster. Less important (below the fold) ones are waiting (max. 2 sec.) for their turn without using bandwidth.

Vanilla JavaScript

Scripts are small and to the purpose. I like pure functions. I don't like inheritance. It doesn't mean I'm about purely functional programming. Mutable state can be useful.

Modular, Easy to Test and Rearrange

All patterns like a headline, a call to action button or a gallery are in a form of modules that can be rearranged. They look good in any order. No need need to write additional code to change a page structure.

Just what's needed

Everything is based on familiar, battle-tested UI patterns. No fancy animations and illustrations that don't do anything except looking cool.
